Check Notifications For Suspended Resumed Jobs

Setting
Documentation Test that notifications are sent for a job which finishes normally.\nIt can be used with both old and new blparser, configuring the right way the testsuite.\n\nSubmit this JDL to a CREAM-CE using the glite-ce-job-submit command:\n\nglite-ce-job-submit -a -r <CEId> test1.jdl\n\ncat test1.jd:\n[\nexecutable="/bin/sleep";\narguments="200";\n]\n\nThen suspends the job with glite-ce-job-suspend.\n\nCheck the batch system and verify than the job is actually suspended. Then wait for 5 minutes and then resume the job using the glite-ce-job-resume command.\n\nAfter the job finishes (it must be needed to then wait for a couple of minutes), check the log file /var/log/cream/glite-xxxparser.log. For the relevant job there should be:\n\n
* A notification with JobStatus=1 (this can be missing)\n
* A notification with JobStatus=2 (this can be missing)\n
* A notification with JobStatus=5 (this must be there)\n
* A notification with JobStatus=2 (this can be missing)\n
* A notification with JobStatus=4 (this must be there) \n\nNote that with PBS and SGE it is possible to suspend only IDLE (i.e. not yet running) jobs.
